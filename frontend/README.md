# 城市医疗管理系统 - 前端

基于 Vue 3 + Vite 构建的城市医疗管理系统前端项目，支持市政管理员、医院管理员和居民（公众用户）三种角色。

## 项目结构

```
src/
├── assets/              # 静态资源
├── components/          # 公共组件
│   ├── common/          # 通用组件（如导航栏）
│   ├── public/          # 公共页面组件
│   └── admin/           # 管理员组件
├── views/               # 页面视图
│   ├── public/          # 公共页面（首页、医院列表、医院详情）
│   ├── admin/           # 管理员主页
│   ├── hospital_admin/  # 医院管理员页面
│   ├── municipal_admin/ # 市政管理员页面
│   └── auth/            # 认证相关页面
├── router/              # 路由配置
├── stores/              # Pinia 状态管理
├── services/            # API 服务
└── utils/               # 工具函数
```

## 功能模块

### 公众端页面
- 首页（系统门户）
- 医院列表页（支持筛选搜索）
- 医院详情页

### 医院管理员后台
- 医院信息管理
- 科室资源管理
- 医护人员管理
- 突发事件上报/参与
- 评分上报

### 市政管理员后台
- 医院管理（增删改）
- 标准科室库管理
- 全市医院资源总览
- 全市医护人员统计
- 突发事件管理
- 等级设置

## API 服务

项目使用 Axios 封装了与后端的 API 通信，支持：

- 公共 API（无需认证）
- 医院相关 API
- 科室相关 API
- 人员相关 API
- 事件相关 API
- 医院等级 API
- 评分相关 API

## 状态管理

使用 Pinia 进行状态管理，包含：

- `auth`：认证状态管理
- `hospital`：医院相关数据管理

## 路由和权限

- 前端路由使用 Vue Router 4
- 实现了基于角色的路由守卫
- 支持市政管理员和医院管理员的权限验证

## 安装和运行

1. 确保已安装 Node.js（版本 >= 16.0.0）
2. 安装依赖：
   ```bash
   npm install
   ```
3. 启动开发服务器：
   ```bash
   npm run dev
   ```
4. 构建生产版本：
   ```bash
   npm run build
   ```

## 开发说明

1. 后端服务需要运行在 `http://localhost:8000`
2. 前端运行在 `http://localhost:5173`
3. 所有 API 请求会自动携带认证令牌（如果存在）
4. 路由守卫会检查用户权限并重定向未授权访问
